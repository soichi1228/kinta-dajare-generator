# 金太ダジャレ製造機

PythonとGemini APIを使用して、「金太」と「キンタマ」の
ぎなた読みによるダジャレを自動生成・評価するプログラムです。

## 概要

「金太、負けが多い」

という文章は、区切り方を変えると、

「キンタマ、毛が多い」

と読むことができます。

このプログラムでは、次の処理を自動的に行います。

1. 読みが「ま」で始まる短文をGeminiで生成する
2. 短文をひらがなへ変換する
3. 先頭の「ま」を削除する
4. 残った読みを別の日本語表現へ変換する
5. 「金太側」と「キンタマ側」の読みが一致するか確認する
6. 日本語としての成立性と面白さをGeminiで評価する
7. 高得点作品をTSVファイルへ蓄積する

## 現在の課題

現在、AIが高得点を付ける作品と、人間が面白いと感じる作品が
一致しないという問題があります。

AIが高く評価したものの、人間にはあまり面白く感じられない例です。

| 金太側 | キンタマ側 |
|---|---|
| まめに掃除をこなす | 目に掃除をこなす |
| 待ちわびる | 血詫びる |
| 豆を砕く | 目を砕く |
| 豆を煎る | 目を射る |
| 豆を土に植えて成長を待つ | 目を土に植えて成長を待つ |

「文章として成立すること」
「奇妙な情景を想像できること」
「人間が笑ってしまうこと」を、どのように区別して評価すべきか
検討しています。

## 必要環境

- Python 3.9以上
- Gemini APIキー

## インストール

リポジトリをダウンロードした後、仮想環境を作成します。

Windowsの場合：

```powershell
python -m venv .venv
.venv\Scripts\activate
python -m pip install -r requirements.txt
```

## Gemini APIキーの設定

Windowsの環境変数に、次の名前でGemini APIキーを設定します。

```text
GEMINI_API_KEY
```

APIキーはPythonファイルや`.env`に直接記載して、
GitHubへ公開しないでください。

環境変数を設定した後は、新しいPowerShellまたは
コマンドプロンプトを開いてください。

## 実行方法

```powershell
python kinta_dajare_generator.py
```

## 主な出力ファイル

実行すると、次のようなファイルが作成されます。

- `kouho.txt`
- `kouho_kana.txt`
- `kouho_kana_no_ma.txt`
- `kouho_henkan.txt`
- `kouho_stage3.tsv`
- `kouho_stage4_v2.tsv`
- `kouho_stage4_history_v2.tsv`
- `kouho_stage4_over80_v2.tsv`
- `kouho_history.tsv`

これらは実行結果や履歴なので、通常はGitHubへ登録しません。

## 使用モデル

初期設定では、次のモデル名を使用しています。

```text
gemini-3.1-flash-lite
```

利用できるモデルが異なる場合は、Pythonファイル内の
`MODEL_NAME`を変更してください。

## 注意事項

このプログラムには、身体部位を題材にした下品な言葉遊びが含まれます。

生成結果が自然な日本語であること、安全であること、
または人間にとって面白いことを保証するものではありません。
